# 전도축제 레크레이션 운영 — 먼저 읽기 📖

청년부 전도축제 2부(레크레이션) 진행 도구입니다. 노트북 한 대를 빔에 연결해 진행하고,
③요리 코너에서는 조원들이 폰으로 정답을 제출합니다. **설치 없이 브라우저만 있으면 됩니다.**

## 🌐 라이브 주소 (이미 배포됨!)
- **시작 페이지(참가자 QR 포함):** https://bon926.github.io/jeondo-festival/
- **운영자 콘솔(빔 화면):** https://bon926.github.io/jeondo-festival/운영자콘솔_강당화면.html
- 참가자는 시작 페이지의 **QR**을 스캔하면 폰 제출 페이지로 들어와요.
> ⚠️ ③요리 실시간 제출은 **`firebase-config.js` 에 Firebase 값을 넣어야** 켜집니다(아래 "준비" 1~2번 + "업데이트" 참고). 넣기 전에도 ①장소·②인물·④소개는 정상 작동해요.

## ✅ 준비 (행사 1~2일 전)

1. **Firebase 만들기** — `Firebase_설정_가이드.md` 따라 하기 (10분, 무료).
   - ⚠️ 테스트 모드는 30일 뒤 잠겨요 → **행사 임박해서** 만드세요.
2. **`firebase-config.js` 채우기** — 복사한 값 5줄을 붙여넣고 저장. (이 한 파일만!)
3. **장소 이름 바꾸기** — 콘솔 → ① 장소 → 우리 교회 실제 장소로 이름 수정/추가/삭제.
4. **(권장) 호스팅** — 아래 "배포" 참고. 짧은 링크 + QR로 나눠주면 제일 편해요.
5. **리허설** — 폰 2~3대로 ③요리 한 바퀴 돌려보기. 끝나면 콘솔 ③요리 → **"③요리 전체 초기화"**.
6. **장소 사진** — 행사 직전 촬영해 ① 장소에서 업로드. (사진은 노트북에 저장돼 새로고침해도 유지)

## 🚀 배포 — 이미 GitHub Pages에 올라가 있어요

주소: **https://bon926.github.io/jeondo-festival/** (GitHub 저장소 `bon926/jeondo-festival`)
- `main` 브랜치에 **푸시할 때마다 자동 재배포**됩니다(약 1분).

### Firebase 설정값 넣기 / 내용 수정하기 (둘 중 편한 방법)
**A. 깃허브 웹에서 바로 수정 (도구 설치 불필요, 제일 쉬움)**
1. https://github.com/bon926/jeondo-festival 접속 → `firebase-config.js` 클릭
2. 연필(✏️) → `"여기에-붙여넣기"` 자리에 Firebase 값 5줄 붙여넣기
3. **Commit changes** → 1분 뒤 라이브에 자동 반영
> 다른 파일(장소 이름, 문제 등)도 같은 방식으로 수정 가능.

**B. 로컬에서 수정 후 푸시**
```bash
# (이 폴더에서)
git add . && git commit -m "firebase 설정" && git push
```

### 행사 끝나면 (정리)
- GitHub 저장소를 **비공개로 바꾸거나 삭제**, Firebase는 **프로젝트 삭제 또는 규칙 잠금**.
  (테스트 모드 DB는 누구나 read/write라 행사 후엔 닫는 게 안전해요.)

> 참고: Netlify에 올리고 싶으면 `npx netlify deploy --prod --dir=.` 도 가능하지만, 이미 위 주소로 라이브라 추가 작업은 필요 없어요.

## ⏱️ 당일 진행 큐시트 (2부, 약 60분)

| 순서 | 코너 | 시간 | 핵심 진행 |
|------|------|------|-----------|
| 0 | 준비 | – | 빔 연결 → `index.html` → 콘솔 열기. 우상단/네비 **● 실시간 연결됨** 확인. ③요리 **전체 초기화** 1회. |
| ⓪ | **⑤ 예측 입력** | 5분 | **게임 시작 전!** 각 조가 ①~④ 우승조를 예측 → **⑤예측** 탭 표에 입력. (행사 끝에 적중 보너스) |
| 1 | ① 장소 맞추기 | 13분 | 사진을 **줌아웃**하며 보여주기 → 외쳐 맞힌 조에 **정답 공개 → 점수**. 빨리 맞출수록 고득점(100/70/40/20). |
| 2 | ② 인물 맞추기 | 13분 | **힌트 공개**(Space)로 한 줄씩 → 맞힌 조에 점수(100/70/40/20). 앞 2명은 연습(목사님 포함). |
| 3 | ③ 요리 협동(폰) | 15분 | **다음 라운드**를 누르면 폰 자동 이동 → 조가 제출 → **✓ 확인 후 점수**(50/40/30). 첫 라운드(라면)는 연습. |
| 4 | ④ 인물 소개 추리 | 13분 | 조가 한 명 정해 힌트 5개 작성(타이머·직접설정) → 한 조씩 공개 → 맞힌 조에 점수. |
| 5 | **⑤ 예측 채점** | 2분 | **⑤예측** 탭 → **결과 채점 & 보너스 지급** → 적중 개수×점수가 ⑤칸에 자동 기록. |
| 끝 | 🏆 최종 발표 | – | 점수판 → **🏆 최종 발표** → "다음 ▶"(Space)로 꼴찌부터 1위까지 공개(컨페티!). |

> 진행 단축키: **Space** = 다음 단계(①줌아웃 / ②④힌트공개 / 최종발표 진행).
> 💡 **✏️ 수동** 버튼(네비)으로 어느 화면에서든 어느 조·칸에든 점수를 직접 주거나 깎을 수 있어요(문제 발생 시 즉시 보정).

## 🎮 코너별 운영 팁

- **① 장소** — 핵심 부분만 클로즈업해 찍으면 줌아웃 재미가 커져요. 사진 없이도 정답 공개로 진행 가능.
- **② 인물** — 성경인물·연예인·위인 섞여 있어요. 너무 쉬우면 운영자가 힌트를 천천히 공개.
- **③ 요리** — ✓(자동 채점)은 **참고용**. 영어/이명 등으로 ✗ 가 떠도 운영자가 맞다고 보면 **"직접 +30점"** 으로 줄 수 있어요. 점수 버튼을 **다시 누르면 취소**돼요. 정답은 한 조가 **한 번만** 내도록 안내(다시 내면 순위가 바뀜).
- **④ 소개** — 힌트 5개 중 1개는 **새가족의 한마디**를 넣게 하면 새가족이 한 번은 발언해요.

## 🏆 점수 규칙
- 합계 = ①장소 + ②인물 + ③요리 + ④소개 + 조정
- 잘못 줬으면: 같은 버튼 다시 눌러 취소하거나, **집계 탭**에서 칸별 −/+ 보정.
- 중간중간 집계 탭 **"💾 점수 내보내기"** 로 백업해두면 안전해요.

## 🛠️ 트러블슈팅

| 증상 | 확인 |
|------|------|
| 폰에 "아직 준비 중" | `firebase-config.js` 값이 채워졌는지 / 파일이 같은 폴더에 있는지 |
| "연결 끊김" 표시 | 인터넷(와이파이/데이터) 확인 |
| 폰 제출이 콘솔에 안 떠요 | ① 인터넷 ② databaseURL 포함 ③ **테스트 모드 30일 만료**(가이드 2번) ④ 콘솔과 폰이 같은 config |
| 조 목록이 폰에 안 떠요 | 콘솔을 한 번 열어 조를 공유했는지 / 인터넷 |
| 점수/사진이 사라졌어요 | 같은 노트북·브라우저면 유지됨. **다른 노트북·시크릿 모드**는 0부터 → 미리 점수 내보내기 |
| QR이 폰에서 안 열려요 | 파일을 직접 연 게 아니라 **호스팅 주소**에서 열어야 함 |

> 가장 흔한 실수: ① 테스트모드 만료, ② config 파일 누락, ③ 콘솔 2대 동시 사용(→ 1대만!).
