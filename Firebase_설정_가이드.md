# 3부 실시간 제출 — Firebase 설정 가이드 (10분, 무료)

3부에서 조별로 폰 정답을 제출하려면, 폰들과 운영자 노트북을 이어줄 **무료 실시간 DB**가 하나 필요해요.
교회당 한 번만 만들면 됩니다. **인터넷(와이파이/데이터)이 되는 환경**에서 진행하세요.

## 1. 프로젝트 만들기
1. https://console.firebase.google.com 접속 → 구글 로그인
2. **프로젝트 추가** → 이름 아무거나(예: `youth-fest`) → 애널리틱스는 꺼도 됨 → 만들기

## 2. 실시간 데이터베이스(Realtime Database) 켜기
1. 왼쪽 메뉴 **빌드 → Realtime Database** → **데이터베이스 만들기**
2. 위치는 기본값, 보안 규칙은 **"테스트 모드로 시작"** 선택 (행사 하루엔 이게 제일 간단)
   - 테스트 모드는 누구나 읽고 쓸 수 있어요. 행사용으론 충분하고, 끝나면 프로젝트를 지우거나 규칙을 잠그면 돼요.
   - ⚠️ **중요 — 테스트 모드는 만든 지 30일이 지나면 자동으로 잠겨(모든 제출 차단) 버려요.** 그러니 프로젝트는 **행사 1~2일 전에** 만드세요. 한 달 넘게 미리 만들어두고 방치하면 행사 당일 폰 제출이 전부 막힙니다. (이미 만료됐다면 Realtime Database → 규칙 탭에서 날짜를 행사일 이후로 늘리거나 `".read"`/`".write"` 를 `true` 로 바꾸세요.)

## 3. 설정값(Config) 복사
1. 왼쪽 위 **⚙️ → 프로젝트 설정**
2. 아래 **내 앱**에서 **웹 앱 추가(</>)** → 앱 이름 입력 → 등록
3. 화면에 나오는 `firebaseConfig = { ... }` 안의 값들을 복사

복사한 값이 이렇게 생겼어요:
```
apiKey: "AIza...."
authDomain: "youth-fest.firebaseapp.com"
databaseURL: "https://youth-fest-default-rtdb.firebasedatabase.app"
projectId: "youth-fest"
appId: "1:1234...:web:abcd..."
```
> ⚠️ `databaseURL` 이 꼭 있어야 해요. 안 보이면 2번(Realtime Database)을 먼저 켰는지 확인하세요.

## 4. `firebase-config.js` 한 파일에만 붙여넣기 (★ 여기만 고치면 됨)
예전엔 두 HTML에 따로 넣었지만, 이제 **`firebase-config.js` 한 곳만** 채우면 콘솔·폰 둘 다 적용돼요.

`firebase-config.js` 를 메모장(또는 편집기)으로 열면 이런 칸이 있어요:
```js
window.FIREBASE_CONFIG = {
  apiKey: "여기에-붙여넣기",
  authDomain: "여기에-붙여넣기",
  databaseURL: "여기에-붙여넣기",
  projectId: "여기에-붙여넣기",
  appId: "여기에-붙여넣기"
};
```
`"여기에-붙여넣기"` 자리에 3번에서 복사한 값을 넣고 저장하세요. 끝!
> ⚠️ 배포(호스팅)하거나 파일을 나눠줄 때는 `firebase-config.js` 도 **반드시 같은 폴더에 함께** 두세요. (이 파일이 없으면 실시간 제출이 안 켜져요.)

## 5. 확인
1. 노트북에서 `운영자콘솔` 열기 → ③요리 탭 → 오른쪽 위 표시가 **● 실시간 연결됨** 이면 성공
2. 폰에서 `폰힌트` 열기 → 조 선택 화면에 운영자가 만든 **조 목록**이 뜨면 연결된 거예요
3. 폰에서 정답 제출 → 콘솔 ③요리의 "실시간 제출 현황"에 바로 뜨는지 확인

## 행사 당일 체크리스트 (3부 시작 전)
1. **콘솔은 노트북 한 대에서만** 여세요. 두 대를 동시에 열면 라운드/조 목록이 서로 덮어써져 꼬여요.
2. ③요리 탭에서 **"③요리 전체 초기화"** 를 한 번 눌러, 리허설 때 들어간 옛 제출을 비우고 1라운드(연습)부터 시작하세요.
3. 강당 빔에 콘솔을 띄우고, 오른쪽 위(또는 상단 네비) 표시가 **● 실시간 연결됨** 인지 확인.
4. 행사 중간중간 집계 탭의 **"💾 점수 내보내기"** 로 점수를 파일 백업해두면, 노트북 사고가 나도 안전해요.

## 운영 흐름 (3부)
1. 운영자: ③요리 탭에서 **다음 라운드 ›** 를 누르면 모든 폰이 그 라운드로 자동 이동
2. 조: 폰 힌트들 보며 의논 → **정답 제출** (조에 하나, 다시 내면 덮어씀)
   - 같은 조가 정답을 다시 제출하면 제출 시각이 갱신돼 **순위(50/40/30)가 바뀔 수 있어요** — 정답은 한 번만 내도록 안내하세요.
3. 운영자: 제출 목록에서 ✓(자동 채점) 확인 → **+점수** 버튼으로 ③요리 칸에 기록
   - 점수는 정답 제출 순서대로 50 / 40 / 30 추천값이 자동 표시돼요
   - ✓ 는 **참고용**이에요. 영어/이명 등으로 ✗ 가 떠도 운영자가 보고 맞다고 판단하면 **"직접 +30점"** 버튼으로 줄 수 있어요.
   - 점수 버튼을 **한 번 더 누르면 취소**돼요(실수로 두 번 줘도 안전).
4. (선택) **정답 공개(화면)** 로 강당 화면에 정답을 크게 띄우기 (폰에는 안 떠요 — 강당 빔 전용)

## 자주 묻는 것
- **여러 교회가 같은 파일을 쓰면 섞이나요?** 각 교회가 자기 Firebase를 만들면(=config가 다르면) 완전히 분리돼요. 한 프로젝트 = 한 행사. 단, **같은 config를 두 곳(예: 다른 방)에서 동시에 열면 섞입니다.** 한 프로젝트는 한 공간에서만 쓰세요.
- **새가족 폰에 앱 설치가 필요한가요?** 아니요. 링크(또는 파일)만 열면 됩니다. 호스팅(예: GitHub Pages, Netlify)에 올려 짧은 링크로 나눠주면 제일 편해요.
- **연결이 안 떠요.** ① 인터넷 연결 확인 ② databaseURL 포함 여부 ③ 두 파일의 config가 동일한지 ④ Realtime Database가 "테스트 모드"인지 ⑤ **테스트 모드가 30일 지나 만료되지 않았는지**(위 2번 ⚠️ 참고) 확인.
- **사진/점수가 사라졌어요.** 점수와 등록한 장소 사진은 그 노트북에 저장돼요(새로고침해도 유지). 단 **다른 노트북/시크릿 모드**로 열면 점수가 0부터예요. 집계 탭의 **점수 내보내기**로 미리 백업하세요.
